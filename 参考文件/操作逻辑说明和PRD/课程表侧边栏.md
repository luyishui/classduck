# 课程表侧边栏 操作逻辑说明与PRD

## 1. 文档目标
- 定义“课程表侧边栏”完整交互、数据逻辑、联动关系与验收标准。
- 作为设计、前端、客户端、测试的统一执行基线。
- 覆盖你提出的 7 个设置分组：
- 课表设置
- 上课时间
- 上午课程
- 下午课程
- 晚上课程
- 周数设置
- 开学日期

## 2. 页面定位与触发入口
- 所属一级页面：课程表主页面。
- 打开方式：点击课程表主页面左上角“三横杠功能键”。
- 呈现方式：左侧抽屉式侧边栏。
- 打开后背景态：
- 原课程表主页面整体降噪（模糊 + 灰色遮罩）。
- 主页面内容可隐约可见，但不可被误触。

## 3. 侧边栏整体结构规范

### 3.1 容器层级
1. 顶层：全屏遮罩层（半透明灰 + 背景模糊）。
2. 左侧：侧边栏主容器（可滑入滑出）。
3. 内部：7 个可折叠分组卡片（抽屉式展开）。

### 3.2 分组卡片视觉规范
- 每个分组为“标题行 + 可展开内容区”。
- 标题行左侧有小图标，内容大容器底色与图标主色同色系。
- 颜色要求：
- 课表设置：淡黄色。
- 上课时间：蓝色。
- 上午课程：橙色。
- 下午课程：与设计稿一致（同类色系区分于上午）。
- 晚上课程：与设计稿一致（夜间语义色）。
- 周数设置：与设计稿一致。
- 开学日期：与设计稿一致。

### 3.3 展开规则
- 默认仅展开 1 个分组（推荐“课表设置”），其他折叠。
- 用户点击任意分组标题：
- 若当前折叠 -> 展开。
- 若当前展开 -> 折叠。
- 分组展开方式：抽屉式下拉内容区（动画平滑）。

### 3.4 关闭规则
- 点击左上角关闭按钮关闭侧边栏。
- 点击遮罩区域关闭侧边栏。
- 侧滑手势（向左）可关闭（可选，但建议支持）。
- 关闭后回到课程表主页面，保持用户先前浏览状态（周次、滚动、激活态）。

## 4. 控件规范（通用）

### 4.1 数值步进控件（加减 + 数字输入）
用于：每节课时长、课间休息时长、上午课程结束、下午课程结束、晚上课程结束、学习周数等。

结构：
- 左：减号按钮。
- 中：白色数字输入容器。
- 右：加号按钮。

交互：
1. 点击加号：+1（或按定义步长）。
2. 点击减号：-1（不低于最小值）。
3. 点击中间数字：进入编辑态，支持直接输入数字。
4. 输入完成触发校验：
- 非法字符过滤。
- 超范围回退到最近合法值。

样式要求：
- 加减按钮色值比外层底框更深，用于层级区分。
- 编辑态白框高亮，失焦后恢复。

### 4.2 旋轴控件（滚轮）
用于：每周起始日、开学日期（年/月/日）。

交互：
- 上下滑动滚轮改变值。
- 中央白色锁定区表示当前选中项。
- 支持循环滚动（星期建议循环，日期按自然边界处理）。

### 4.3 时间输入控件
用于：各节次的开始/结束时间。

交互：
- 点击时间值进入输入态。
- 输入规则：
- 小时范围 00-23。
- 分钟范围 00-59。
- 自动补零展示（如 8:5 -> 08:05）。

## 5. 七大设置分组详细逻辑

## 5.1 课表设置（淡黄色）

### 5.1.1 功能目标
- 管理多个课表（新建、切换、当前课表标识）。

### 5.1.2 新建课表
- 点击“新建课表”后，弹出小白色输入容器（含勾/叉）。
- 勾：确认创建。
- 叉：取消创建。

创建规则：
1. 名称不能为空。
2. 名称长度限制（建议 1-20）。
3. 可重名策略：
- 建议允许但自动补后缀，或直接提示重名。

### 5.1.3 多课表切换
- 课表列表显示当前已有课表。
- 被选中课表右侧显示小勾。
- 点击其他课表立即切换为当前课表。

### 5.1.4 与课程表主页面联动
- 切换后主页面数据源切到对应 tableId。
- 主页面课程块、周视图、时间轴全部更新为该课表数据。
- 若目标课表无课程，显示空态引导。

### 5.1.5 验收点
- 新建成功后立刻出现在列表。
- 当前课表勾选唯一。
- 切换后主页面课程内容同步变化。

## 5.2 上课时间（蓝色）

### 5.2.1 功能目标
- 定义全局课时模板：
- 每节课时长（分钟）
- 课间休息时长（分钟）

### 5.2.2 控件逻辑
- 两项均使用“加减 + 数字输入”控件。

数值约束建议：
- 每节课时长：10-180 分钟。
- 课间休息时长：0-120 分钟。

### 5.2.3 一键批量对齐（时间吸附）
- 按钮含义：根据“每节课时长 + 课间休息时长”重新推算后续节次时间。

示例：
- 原模板：45 分钟课时 + 10 分钟课间。
- 第 1 节：08:00-08:45，第 2 节默认 08:55 开始。
- 用户改为：30 分钟课时 + 10 分钟课间。
- 点击“一键批量对齐”后：第 2 节自动吸附为 08:40 开始。

公式：
- n+1 节开始时间 = n 节结束时间 + 课间休息时长。
- n 节结束时间 = n 节开始时间 + 每节课时长。

### 5.2.4 与上午/下午/晚上课程联动
- 上课时间修改后：
- 若用户点击一键批量对齐，上午/下午/晚上节次时间应整体重算。
- 若未点击批量对齐：
- 可采用“待对齐状态”提示（建议），避免用户误解。
- 确认提交后，至少要保证模板层数据已更新。

### 5.2.5 与主页面联动
- 主页面左侧时间轴立刻使用最新模板。
- 课程卡片纵向占位高度可按节次映射，不强制受分钟变化影响（由渲染策略决定）。

### 5.2.6 验收点
- 修改时长后，批量对齐能正确吸附后续时间。
- 主页面节次时间同步更新。
- 非法输入被拦截并提示。

## 5.3 上午课程（橙色）

### 5.3.1 功能目标
- 设置上午课程节次数量与每节具体时间。

### 5.3.2 上午课程结束
- 控件：加减 + 数字输入。
- 语义：上午最后一节是第 N 节。

规则：
1. 当 N 从 4 改为 3：仅显示第 1-3 节编辑项。
2. 当 N 从 4 改为 6：显示第 1-6 节编辑项。

### 5.3.3 节次时间编辑
- 每节展示“开始时间 - 结束时间”。
- 点击时间值可输入精确数字调整。

### 5.3.4 与上课时间联动（强锚定）
- 上课时间中的每节课时长变化后：
- 上午各节默认时长应同步变化。
- 在批量对齐后，上午各节时间必须反映最新模板。

### 5.3.5 与下午课程衔接
- 下午第一节编号 = 上午最后一节 + 1。
- 上午结束节次变化会直接影响下午起始编号。

### 5.3.6 与主页面联动
- 主页面上午区段节次数量、时间轴、课程映射即时更新。

### 5.3.7 验收点
- 上午结束节次变化后，节次列表数量正确增减。
- 上午每节时间可独立编辑并生效。
- 调整上午结束后，下午首节编号自动重排。

## 5.4 下午课程

### 5.4.1 功能目标
- 设置下午课程结束节次和下午各节时间。

### 5.4.2 关键规则
- 下午节次起点严格依赖上午结束节次。
- 示例：上午最后一节=第4节，则下午从第5节开始。

### 5.4.3 编辑逻辑
- 与上午课程一致：结束节次 + 各节时间编辑。
- 时间可手动输入，支持精确到分钟。

### 5.4.4 与晚上课程衔接
- 晚上第一节编号 = 下午最后一节 + 1。

### 5.4.5 与主页面联动
- 主页面下午区段节次、时间轴标签同步变化。

### 5.4.6 验收点
- 下午首节编号始终正确承接上午末节。
- 调整下午结束后，晚上首节编号自动更新。

## 5.5 晚上课程

### 5.5.1 功能目标
- 设置晚上课程结束节次和各节时间。

### 5.5.2 关键规则
- 晚上节次起点严格依赖下午结束节次。
- 示例：下午最后一节=第8节，则晚上从第9节开始。

### 5.5.3 编辑逻辑
- 与上午/下午一致：结束节次 + 每节时间编辑。

### 5.5.4 与主页面联动
- 主页面晚间区段节次、时间轴、课程对应关系同步。

### 5.5.5 验收点
- 晚上首节编号承接关系正确。
- 每节时间修改后主页面即时更新。

## 5.6 周数设置

### 5.6.1 功能目标
- 定义学期总周数与每周起始日。

### 5.6.2 学习周数
- 控件：加减 + 数字输入。
- 语义：该课表学期总周数（如 18、20）。

约束建议：
- 最小 1，最大 35。

### 5.6.3 每周起始日
- 控件：旋轴（星期一至星期日）。
- 中央白色容器锁定选中项。
- 支持上下滑动快速切换。

### 5.6.4 与主页面联动
- 主页面周视图首列 = 选定起始日。
- 当起始日从周一改为周日时：
- 主页面第一列应变为周日，后续列顺序整体平移。

### 5.6.5 验收点
- 周数变更后，周次上限计算正确。
- 起始日变更后，主页面列顺序正确。

## 5.7 开学日期

### 5.7.1 功能目标
- 定义“第 1 周起点日期”。

### 5.7.2 控件结构
- 年/月/日三个短滚轮拼成一个整体日期选择器。

### 5.7.3 展示规则
- 选定后，分组标题右侧展示当前日期（如 2024-09-02）。

### 5.7.4 与主页面联动
- 用于计算“当前周次”。
- 公式示意：
- 当前周次 = floor((今日日期 - 开学日期) / 7) + 1。
- 与“每周起始日”共同决定周边界。

### 5.7.5 边界处理
- 若今日日期 < 开学日期：可显示“未开学”或第 0 周（按产品策略）。
- 若超出学习周数：可显示“已结课”或最后一周（按产品策略）。

### 5.7.6 验收点
- 日期选择后标题区有回显。
- 当前周次计算与预期一致。

## 6. 设置之间的互相关系（核心联动矩阵）

### 6.1 时间模板链
- 上课时间（课时/课间）是时间模板源头。
- 上午/下午/晚上的每节时间是模板的落地结果。
- 批量对齐是模板向节次时间的同步动作。

### 6.2 节次编号链
- 上午结束决定下午起点。
- 下午结束决定晚上起点。
- 任一段结束变化都触发后续分段编号重排。

### 6.3 周历链
- 开学日期 + 每周起始日 + 学习周数 -> 周次系统。
- 周次系统决定主页面显示逻辑（当前周、首列、可滑动范围）。

### 6.4 课表切换链
- 课表设置切换 tableId 后：
- 上述所有配置（时间模板、周设置、节次分段）切换到目标课表版本。

## 7. 与课程表主页面的同步策略

### 7.1 同步时机
- 推荐实时同步：每次设置确认后即时写入并刷新主页面。
- 输入中间态（未确认）可暂存，不立即影响主页面。

### 7.2 同步范围
1. 时间轴：节次时间、节次数量。
2. 周视图：首列起始日、周次计算。
3. 课表数据：tableId 切换后的课程集合。

### 7.3 刷新策略
- 局部刷新优先，避免整页重建。
- 切换课表可全量刷新。

## 8. 数据模型建议

### 8.1 CourseTableConfig（按课表存储）
- tableId: number
- classDurationMin: number
- breakDurationMin: number
- morningEndSection: number
- afternoonEndSection: number
- eveningEndSection: number
- sectionTimeList: array
- termWeeks: number
- weekStartDay: enum(Mon...Sun)
- semesterStartDate: yyyy-mm-dd

### 8.2 sectionTimeList 示例
- [{section:1,start:'08:00',end:'08:45'}, ...]

## 9. 事件定义（埋点建议）
- onOpenScheduleSidebar
- onCloseScheduleSidebar
- onToggleSectionDrawer(sectionKey)
- onCreateScheduleConfirm
- onSwitchSchedule(tableId)
- onChangeClassDuration(value)
- onChangeBreakDuration(value)
- onClickBatchAlign
- onChangeMorningEnd(value)
- onChangeAfternoonEnd(value)
- onChangeEveningEnd(value)
- onChangeSectionTime(section, start/end)
- onChangeTermWeeks(value)

## 10. 异常与容错
1. 非法数字输入：
- 拦截并提示“请输入有效数字”。
2. 时间反转（结束早于开始）：
- 阻止保存并高亮错误项。
3. 分段节次冲突（上午结束 >= 下午结束）：
- 阻止保存并提示分段顺序错误。
4. 课表切换失败：
- 回退到原课表并提示失败。

## 11. 动效与反馈规范
- 侧边栏：300ms 左右滑入滑出。
- 抽屉展开：200-250ms 高度过渡。
- 保存成功：轻提示（toast/snackbar）+ 主页面局部刷新。

## 12. 全量验收标准（强执行）

### 12.1 入口与基础态
1. 点击主页面左上角三横杠后，侧边栏可正常打开。
2. 打开后背景出现灰色遮罩与模糊，主页面可见但不可误触。
3. 点击遮罩可关闭侧边栏。

### 12.2 抽屉分组
4. 7 个分组全部存在且可展开/折叠。
5. 分组底色与图标色系一致。
6. 抽屉展开内容区为大容器样式，内含各小设置项。

### 12.3 课表设置
7. 新建课表弹出白色输入容器，勾/叉行为正确。
8. 多课表可切换，当前课表后有唯一小勾。
9. 切换课表后主页面课表内容同步。

### 12.4 上课时间
10. 每节课时长与课间时长支持加减和手输。
11. 一键批量对齐后，节次时间按模板吸附更新。
12. 主页面时间轴同步更新。

### 12.5 上午/下午/晚上课程
13. 上午结束节次变化时，上午节次编辑项数量正确变化。
14. 下午首节编号始终 = 上午末节 + 1。
15. 晚上首节编号始终 = 下午末节 + 1。
16. 各节开始/结束时间可编辑并即时生效。
17. 上午/下午/晚上任一分段修改后主页面对应区段同步。

### 12.6 周数设置
18. 学习周数支持加减和手输，范围校验正确。
19. 每周起始日滚轮可上下切换并正确锁定选中项。
20. 起始日变更后主页面第一列同步变化。

### 12.7 开学日期
21. 年/月/日三滚轮可独立调节。
22. 选中日期后标题旁回显日期。
23. 周次计算基于开学日期生效，主页面当前周正确。

### 12.8 跨模块联动
24. 任意设置项确认后，主页面对应数据在可接受时延内刷新（建议 < 300ms）。
25. 关闭并重新打开侧边栏后，设置状态保持。
26. 切换课表后，每个课表各自配置独立保存，不串线。

### 12.9 稳定性
27. 连续快速操作（加减、展开折叠、切换课表）无崩溃。
28. 非法输入不导致主页面异常渲染。
29. 日期边界（闰年、月末）处理正确。

## 13. 研发实现建议（可选）
- 配置以 tableId 为主键独立持久化。
- 批量对齐执行前先快照，支持撤销（可选增强）。
- 把联动计算抽为纯函数，便于单测覆盖。

## 14. 回归测试重点
1. 课时改动 + 批量对齐 + 主页面时间轴。
2. 上午结束改动对下午首节编号影响。
3. 起始日由周一改周日后的主页面列顺序。
4. 开学日期变更后的当前周次计算。
5. 多课表切换后的配置隔离。
