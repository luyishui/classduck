# 任务16-导入引擎落库链路接入

## 1. 任务目标
将导入模块从“仅页面占位 + 日志上报”推进到“可执行导入并写入课程库”的状态，形成阶段E的核心通路雏形。

## 2. 任务范围
- 新增导入领域模型（ImportCourse / ImportTable）
- 新增 ImportEngine（执行解析 -> 创建课表 -> 写课程）
- 导入执行页接入 ImportEngine
- 失败路径保留日志上报
- 静态检查验证

## 3. 改动文件清单与作用说明
1. `classduck_app/lib/features/import/domain/import_course.dart`
- 导入课程领域模型。

2. `classduck_app/lib/features/import/domain/import_table.dart`
- 导入课表领域模型。

3. `classduck_app/lib/features/import/application/import_engine.dart`
- 导入引擎主流程：
- 解析结果构建（当前为过渡 mock）
- 课表创建
- 课程批量写入
- 返回导入结果（课表ID、课程数）

4. `classduck_app/lib/features/import/ui/import_execution_page.dart`
- 增加“Run Import and Save Courses”执行按钮。
- 导入成功展示结果。
- 导入失败触发 `/v1/import/logs` 上报。

## 4. 核心函数说明
### 4.1 `ImportEngine.importFromSchoolConfig(config)`
- 输入：学校配置 `SchoolConfig`
- 输出：`ImportExecutionResult`
- 逻辑：
1. 调用解析执行步骤（当前 mock）
2. 创建课程表
3. 映射并写入课程列表
4. 返回导入统计

### 4.2 `ImportEngine._executeParser(config)`
- 输入：学校配置
- 输出：`ImportTable`
- 逻辑：当前为过渡实现，模拟解析结果。
- 说明：后续替换为 WebView + JS 解析。

### 4.3 `ImportExecutionPage._runImport()`
- 输入：用户点击事件
- 输出：UI 执行状态与结果文案
- 逻辑：
1. 置为 importing
2. 调用 ImportEngine
3. 成功显示导入结果
4. 失败显示错误并上报日志

## 5. 架构设计说明
1. 导入流程分层
- UI 层：发起动作与状态展示
- Application 层：导入流程编排
- Data 层：复用课程仓储写库

2. 失败可观测性
- 导入失败时统一走日志仓储，便于后端追踪。

3. 可替换性
- 解析步骤单独抽离，后续替换为真实 WebView 执行无需重写写库流程。

## 6. 验收标准
1. 导入执行页存在“Run Import and Save Courses”按钮。
2. 点击后可走导入流程并返回导入结果文案。
3. 异常路径可触发日志上报。
4. `flutter analyze` 无静态错误。

## 7. 验收结果
- 改动完成并通过静态检查。
