# 任务30：侧边栏新建类型联动与外观主题页重构

## 1. 目标
1. 修复待办侧边栏交互语义：将“新建待办”调整为“新建类型”，移除抽屉内内联输入区，改为弹窗创建类型。
2. 打通“新建待办”页的任务类型联动：从待办主页动态传入当前类型列表。
3. 修复“无课程时仍强制关联课程”的体验问题：无课程时隐藏关联课程区并取消强制校验。
4. 按 PRD 重构“外观与主题”二级页（中文文案、浅/深模式、背景图上传/清除入口）。

## 2. 变更文件
- `classduck_app/lib/features/todo/ui/todo_page.dart`

## 3. 关键实现
### 3.1 待办侧边栏
1. 顶部改为居中的“鸭 + 上课鸭”结构。
2. 快捷入口从 `+ 新建待办` 改为 `+ 新建类型`。
3. 新建类型由 `_createNewType()` 统一弹窗处理，创建后同步写入 `_typeOrder/_typeLabels`。

### 3.2 新建待办联动
1. `NewTodoPage` 新增构造参数：`taskTypeLabels`、`courseOptions`。
2. 页面内部 `_taskTypes` 由传入 map 动态构建，默认选中第一个类型。
3. 仅当 `courseOptions` 非空时展示“关联课程”模块，并在提交时做课程必填校验。

### 3.3 外观与主题页
1. 页面改为中文移动端样式，顶栏标题居中“外观与主题”。
2. 模式切换保留浅色/深色两卡片，选中态为浅黄色容器。
3. 新增“上传背景图片/清除背景图片”操作按钮与当前状态提示。

### 3.4 补充修复
1. `manual_add_course_page.dart` 颜色弹窗引用修复，改用本页 `_colorPalette` 避免未导入符号。
2. `main.dart` 清理不必要 import，保持数据库初始化代码通过分析。
