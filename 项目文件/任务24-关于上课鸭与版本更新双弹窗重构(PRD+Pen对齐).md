# 任务24-关于上课鸭与版本更新双弹窗重构（PRD+Pen对齐）

## 1. 任务目标
将“关于上课鸭”页面从旧版简化实现重构为 PRD + Pen 对齐版本，覆盖以下范围：
1. 顶栏、关于卡片、底部协议与备案信息。
2. 行为项：小红书复制、隐私政策入口、版本更新入口。
3. 版本更新双弹窗：
- 已是最新（检查更新结果）
- 发现新版本（含可滚动更新说明与双按钮）
4. 保持弹窗统一关闭规则（遮罩可关闭、双击空白可关闭）。

## 2. 本次改动文件
1. `classduck_app/lib/features/settings/ui/about_page.dart`（重写）
- 将英文默认页替换为中文 PRD 结构。
- 新增关于卡片三行布局与对应交互。
- 新增版本检测结果模型与两种弹窗实现。

2. `classduck_app/lib/shared/widgets/duck_modal.dart`（增强）
- `DuckModal.show` 新增可选参数：`barrierColor`、`blurSigma`。
- 保持原有调用兼容，同时允许页面按 Pen 配置遮罩透明度。

## 3. 关键实现说明
### 3.1 页面结构与视觉
在 `AboutPage` 中完成以下对齐：
1. 页面背景 `#FFFDF8`。
2. 顶栏标题“关于上课鸭”，左侧返回箭头。
3. 主卡片圆角 28，阴影参数对齐 Pen（外阴影、偏移 0/18、模糊 40）。
4. 三行入口：
- 小红书（右侧“复制”黄色按钮）
- 隐私政策（右箭头）
- 版本更新（右侧版本号 + 右箭头）
5. 底部文案：服务协议/隐私政策 + ICP 备案号。

### 3.2 交互函数
1. `_copyRedBook()`
- 使用 `Clipboard.setData` 写入文本。
- 成功后通过 `SnackBar` 提示“已复制”。

2. `_openPrivacyPolicy()`
- 当前先给出占位提示（后续可接入正式路由页面）。

3. `_checkUpdate()`
- 调用 `_mockCheckUpdate()` 获取版本比较结果。
- 根据 `hasNewVersion` 分流：
- true -> `_showFoundNewVersionModal()`
- false -> `_showLatestModal()`

4. `_showLatestModal()`
- 弹窗尺寸 336x196。
- 标题“检查更新”、正文“当前已是最新版本”、按钮“我知道了”。

5. `_showFoundNewVersionModal()`
- 弹窗尺寸 336x268。
- 标题、副标题、74px 高度滚动更新说明区。
- 按钮“暂不更新 / 立即更新”。

### 3.3 组件化拆分
1. `_AboutActionRow`
- 统一三行入口布局，支持图标、标题、右值、右侧自定义区域和点击回调。

2. `_VersionCheckResult`
- 维护当前版本/最新版本/更新说明。
- 通过 `currentVersion != latestVersion` 计算是否有新版本。

3. `_RedBookDot`
- 提供小红书行的简化红色图标占位，避免依赖外部资源。

## 4. 与 PRD + Pen 对齐点
1. 页面文案已全部中文化，语义与 PRD一致。
2. 关于卡片、圆角、阴影、分割线、底部文本布局按 Pen 结构重建。
3. “已是最新”与“发现新版本”两套弹窗均按 Pen 尺寸与文案结构实现。
4. 更新说明区采用灰色容器 + 可滚动正文，符合 PRD 对更新内容区的要求。
5. 弹窗遮罩采用 `#00000066`，并保持双击空白关闭规则。

## 5. 验收标准
1. 进入“关于上课鸭”页面后，视觉结构与 PRD/Pen 一致（顶栏、三行卡片、底部备案）。
2. 点击“小红书-复制”会复制文本并出现提示。
3. 点击“版本更新”可弹出“发现新版本”弹窗，且说明区可滚动。
4. 两种弹窗按钮行为正确：
- 暂不更新/我知道了：关闭弹窗
- 立即更新：关闭弹窗并提示更新流程（占位）
5. 静态检查无错误。
