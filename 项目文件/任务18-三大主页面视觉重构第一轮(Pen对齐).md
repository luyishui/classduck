## 4. 核心函数设计说明
### 4.1 课程表页面
- `_buildScheduleGrid()`
  - 输入：当前课程列表 `_courses`
  - 输出：带节次轴与周列网格的 UI
  - 逻辑：先绘制网格线，再按课程 `weekTime/startTime/timeCount` 进行定位绘制

- `_buildCourseCard(...)`
  - 输入：单条课程与网格尺寸
  - 输出：定位后的课程色块卡片
  - 逻辑：将课程时间映射为 `Positioned` 的 `left/top/height`

- `_currentWeekDates(now)`
  - 输入：当前日期
  - 输出：本周一到周日日期数组
  - 用途：驱动周头部日期展示

### 4.2 待办页面
- `_buildTodoTab(...)`
  - 输入：任务列表与完成态
  - 输出：按类型分组的列表视图
  - 逻辑：循环类型顺序，渲染标题与对应任务卡片

- `_buildGroupItems(...)`
  - 输入：分组类型、展开状态
  - 输出：该组任务组件列表
  - 逻辑：收起态截断前3条，展开态显示全部

- `_toggleCompleted(item, completed)`
  - 输入：任务ID与目标状态
  - 输出：更新数据库后刷新页面
  - 逻辑：调用 repository 更新完成态，随后重载数据

### 4.3 我的页面
- `_loadStats()`
  - 输入：课程仓储、待办仓储
  - 输出：三个统计字段更新
  - 逻辑：读取课程总数、已完成待办数，并更新状态

- `_openMoodModal()`
  - 输入：用户选中 mood 文案
  - 输出：主页“今日状态”即时更新
  - 逻辑：弹窗内点击选项 -> setState -> 关闭弹窗
