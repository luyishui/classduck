# 任务10-课程表页面接入SQLite真实数据渲染

## 1. 任务目标
把课程表页面从占位展示升级为“读取本地 SQLite 数据并渲染”的可运行形态，为后续导入写库后即时展示打基础。

## 2. 任务范围
- 在课程表页接入 `ScheduleRepository`
- 页面启动时加载课表与课程数据
- 空库场景自动初始化一份演示课表与课程
- 页面提供课程数据刷新入口
- 保留导入入口与配置状态区域

## 3. 本次改动文件清单与作用说明
1. `classduck_app/lib/features/schedule/ui/schedule_page.dart`
- 增加本地数据加载状态管理。
- 增加课表初始化逻辑（空库自动种子数据）。
- 将“网格占位”改为基于课程数据的节次渲染。
- 保留并并行展示“远端配置状态”和“本地课表状态”。

## 4. 核心函数说明
### 4.1 `_bootstrap()`
- 输入：无
- 输出：`Future<void>`
- 逻辑：
1. 加载远端学校配置
2. 加载本地课表数据
- 目的：页面首屏同时具备导入配置状态和课表数据状态。

### 4.2 `_loadScheduleData()`
- 输入：无
- 输出：`Future<void>`
- 逻辑：
1. 查询课表列表
2. 若空库则创建默认课表并写入演示课程
3. 读取激活课表课程列表
4. 更新页面状态

### 4.3 节次渲染逻辑
- 输入：`_courses` 列表
- 输出：10 行节次展示
- 逻辑：
- 按 `startTime` 过滤当前节次课程
- 无课程显示 `free slot`
- 有课程显示课程名拼接

## 5. 架构设计说明
1. 数据分离
- 页面不直接操作数据库，通过 `ScheduleRepository` 访问。

2. 状态分离
- 远端配置状态：`_loadingConfig/_configError`
- 本地课表状态：`_loadingSchedule/_scheduleError`

3. 兼容导入链路
- 后续导入写库后，调用 `_loadScheduleData()` 即可刷新课程视图。

## 6. 执行方案与关键决策
