# 任务36：侧边栏联动规则与适配方案说明

## 1. 本轮实现
1. 课程表侧边栏节次时间调整新增联动规则：
   - 改上课时间时，下课时间自动按课时长度重算；
   - 改当前节次后，后续节次按课间时长和课时长度顺延。
2. 节次时间编辑增加合法性拦截：开始时间必须早于结束时间。
3. 节次时间 UI 调整为灰色小容器样式，点击可修改。
4. 步进控件中间白色数字支持点击手输（带范围夹取）。
5. 新建待办课程改为可选；选择样式改为带勾视觉；按钮黄色做浅化处理。
6. 手动添加课程“更多颜色”弹窗改为完整色板（不再与首页6色重复）。

## 2. 适配规则现状（仅说明，不导入）
1. 后端采用规则文件驱动：`backend/src/config/adapter-rules.sample.json`。
2. 单条规则包含：
   - `match.hostIncludes` / `match.pathIncludes`：匹配目标网站；
   - `extract.scriptUrl`：对应解析脚本地址；
   - `delaySeconds` / `needsLogin`：执行策略。
3. 后端服务 `adapterRuleService.js` 读取规则文件并通过 `/v1/config/adapters` 提供给前端。
4. 前端 `AdapterRuleRepository` 拉取后反序列化为 `AdapterRule`，可在导入流程中按 host/path 选择解析器。
5. `参考文件/aishedule-master` 的结构是 `provider + parser + timer`：
   - `provider` 负责抓取原始页面内容；
   - `parser` 负责提取课程字段；
   - `timer` 负责节次时间模板。

## 3. 备注
- 按用户要求：本轮只说明适配机制，不执行导入动作。
