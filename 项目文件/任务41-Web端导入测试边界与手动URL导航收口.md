# 任务41：Web端导入测试边界与手动URL导航收口

## 1. 背景
1. 用户本轮在 Web（Chrome）端进行导入链路点测。
2. 导入功能的核心抓取依赖内嵌 WebView，在 Web 端存在平台能力限制。
3. 用户明确提出：不要自动跳网页，保留网址导航，让用户可手动改 URL 再触发。

## 2. 目标
1. 收敛导入执行页交互：从“自动/强引导”改为“用户手动触发”。
2. 保留可编辑登录 URL，支持调试或切换入口地址。
3. 明确 Web 端测试边界，防止进入不兼容的内嵌抓取路径导致崩溃。

## 3. 实施内容
- 文件：`classduck_app/lib/features/import/ui/import_execution_page.dart`
- 变更：
1. 新增可编辑登录 URL 输入框（默认填充学校配置 `initialUrl`）。
2. “进入内嵌登录并抓取”改为严格手动触发。
3. Web 端（`kIsWeb`）点击抓取按钮时，仅提示边界说明，不进入内嵌抓取页面。
4. 保留抓取状态展示（未抓取/已抓取时间、抓取页 URL）。
5. 修正日志脱敏正则写法，避免分析器报错。

## 4. 文档同步
1. `说明文档/03-设计修改记录.md` 新增：
- 导入执行页手动触发策略。
- Web 端抓取边界提示策略。

## 5. 当前测试口径（给产品/验收）
1. Web 端可测：学校列表、分栏、搜索、导入执行页静态交互、URL编辑。
2. Web 端不可测：内嵌 WebView 登录抓取。
3. 抓取导入完整闭环请在 Android/iOS 设备验收。

## 6. 结果
1. 避免了 Web 端导入页因 WebView 不兼容导致的崩溃。
2. 用户可自主控制 URL 与触发时机，符合“先测功能、再扩场景”的节奏。
