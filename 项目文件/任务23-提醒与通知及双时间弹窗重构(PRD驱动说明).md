# 任务23-提醒与通知及双时间弹窗重构（PRD驱动说明）

## 1. 任务目标
将“提醒与通知”从通用系统组件页面重构为 PRD 对齐页面，重点覆盖：
1. 胶囊式开关交互（每日提醒、课前提醒，彼此独立）。
2. 课前提醒时间弹窗（3 选项 + 确认）。
3. 每日提醒滚轮弹窗（前1天/前2天 + 小时/分钟滚轮 + 确认）。
4. 关闭与提交行为：仅确认提交，关闭不提交。

## 2. 本次改动文件
1. `classduck_app/lib/features/settings/ui/notifications_page.dart`（重写）
- 重构页面结构、开关样式、时间入口行样式。
- 重构两个时间弹窗，替换旧版系统对话框组合。

## 3. 改动前后变化
### 3.1 页面层
改动前：
- 英文文案，`SwitchListTile + ListTile` 组合。
- 视觉风格偏系统默认，未体现 PRD 胶囊开关和卡片布局。

改动后：
- 中文文案与 PRD 语义一致。
- 两个提醒卡片均为圆角白卡片。
- 开关改为胶囊推拉（黄色开启、灰色关闭、白色圆点左右位移），动画 200ms。
- “提醒时间”行为入口与开关在同一卡片内，右侧 chevron 对齐。

### 3.2 课前提醒弹窗
改动前：
- `SimpleDialog`，布局和样式与 PRD 不一致。

改动后：
- 使用统一 `DuckModal` 弹窗容器。
- 标题“设置提醒时间”。
- 三个选项：课前10分钟/课前30分钟/课前1小时。
- 下方确认按钮，确认后回写。
- 遮罩关闭、返回关闭、双击空白关闭均不提交。

### 3.3 每日提醒滚轮弹窗
改动前：
- `AlertDialog + TimePicker` 组合，不是滚轮结构。

改动后：
- 使用 `CupertinoPicker` 实现小时/分钟双列滚轮。
- 中间固定冒号，选中行配黄色高亮窗口。
- 上方支持“前1天/前2天”切换。
- 确认后回写为“前 X 天 HH:mm”。
- 关闭不提交临时值。

## 4. 核心函数说明
### 4.1 `_openClassReminderModal()`
- 用途：打开课前提醒选项弹窗。
- 关键点：
  - 本地变量暂存选项。
  - 点击“确认”才写回 `_classReminder`。

### 4.2 `_openDailyWheelModal()`
- 用途：打开每日提醒滚轮弹窗。
- 关键点：
  - `FixedExtentScrollController` 管理小时/分钟当前项。
  - 弹窗内 `StatefulBuilder` 管理临时值。
  - 点击“确认”后写回 `_dailyOffsetDay/_dailyHour/_dailyMinute`。

### 4.3 `_PillSwitch`
- 用途：复用胶囊开关组件。
- 关键点：
  - `AnimatedContainer + AnimatedAlign` 做推拉位移动画。
  - 开关与颜色状态完全由 `value` 驱动。

### 4.4 `_ReminderCard`
- 用途：统一提醒卡片结构。
- 关键点：
  - 顶部标题 + 开关。
  - 底部“提醒时间”可点击行。
  - 保证右侧箭头与上方开关右缘接近对齐。

## 5. 与 PRD 的对齐说明
本任务已落实 PRD 的核心逻辑：
1. 两个开关独立。
2. 开关视觉与动画方向符合胶囊逻辑。
3. 两种提醒时间弹窗结构按 PRD 重构。
4. 仅确认提交，关闭不提交。
5. 提交后回写到提醒时间文案。
