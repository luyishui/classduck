# 任务02-安全规范落地与参考资料隔离

## 1. 任务目标
落实项目安全边界，确保后续开发仅在当前工作区内执行，并将所有参考输入集中到统一目录，避免误改、误删和运行时耦合。

## 2. 任务范围
- 更新总实施方案中的安全规范。
- 更新总实施方案中的参考资料目录规范。
- 创建 `参考文件/` 目录。
- 迁移 PRD、Pen、参考代码到 `参考文件/`。

## 3. 本次改动文件清单与作用说明
1. `上课鸭-总实施方案.md`
- 新增“安全规范（强制）”章节。
- 新增“参考资料目录规范”章节。
- 将输入路径统一改为 `参考文件/` 下的路径。
- 补充代码可读性与注释要求。

2. `参考文件/操作逻辑说明和PRD/`
- 作为产品与交互唯一参考源目录。

3. `参考文件/上课鸭.pen`
- 作为视觉稿结构参考源文件。

4. `参考文件/aishedule-master/`
- 作为教务适配规则样本参考目录。

- 沉淀当前任务全过程与验收标准。

## 4. 执行逻辑与关键命令说明
### 4.1 目录调整执行步骤
1. 新建 `参考文件/`。
2. 迁移 `操作逻辑说明和PRD/`、`上课鸭.pen`、`aishedule-master/`。
3. `Class-Shedule-Flutter/` 首次直接移动时遇到访问拒绝。
4. 采用“复制后删除原目录”策略：
- 使用 `robocopy` 递归复制。
- 校验复制成功后删除源目录。

### 4.2 异常处理说明
- 问题：目录被占用导致 `Move-Item` 失败。
- 策略：改用更稳定的目录复制并二次清理。
- 结果：迁移完成，源目录已删除。

## 5. 架构与安全设计说明
1. 物理隔离
- 新项目目录：`classduck_app/`
- 参考目录：`参考文件/`
- 文档目录：`项目文件/`

2. 依赖隔离
- 新项目运行时不依赖 `参考文件/`。
- 参考内容仅用于设计与实现推导，不进入运行时路径。

3. 风险控制
- 通过统一参考目录减少误操作面。
- 通过任务级文档确保每次变更可追踪。

## 6. 验收标准
1. 工作区根目录仅保留以下主目录：
- `classduck_app/`
- `参考文件/`
- `项目文件/`

2. `参考文件/` 下包含：
- `操作逻辑说明和PRD/`
- `上课鸭.pen`
- `aishedule-master/`
- `Class-Shedule-Flutter/`

3. 总实施方案中包含：
- 工作区安全边界
- 参考资料目录规范
- 代码可读性与注释要求
