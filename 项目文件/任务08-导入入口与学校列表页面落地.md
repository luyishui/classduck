# 任务08-导入入口与学校列表页面落地

## 1. 任务目标
实现导入链路的前端入口能力：
1. 从课程表页面进入导入流程。
2. 展示后端学校配置列表。
3. 支持加载、失败、重试、下拉刷新等基础交互状态。

## 2. 任务范围
- 新增学校列表页面
- 将课程表加号入口接入学校列表页
- 列表页面接入 `SchoolConfigRepository`
- 提供学校配置预览对话框
- 执行静态检查

## 3. 本次改动文件清单与作用说明
1. `classduck_app/lib/features/import/ui/import_school_list_page.dart`
- 新建导入学校列表页面。
- 负责加载学校配置、展示状态、支持重试与刷新。
- 点击条目显示配置预览。

2. `classduck_app/lib/features/schedule/ui/schedule_page.dart`
- 将右下角加号从占位弹窗改为页面跳转。
- 入口目标：`ImportSchoolListPage`。

## 4. 核心函数说明
### 4.1 `_load()`（`import_school_list_page.dart`）
- 输入：无
- 输出：`Future<void>`
- 逻辑：
1. 设置 loading 状态
2. 调用 repository 读取配置
3. 更新列表或错误
4. 关闭 loading

### 4.2 `_showConfigPreview(SchoolConfig config)`
- 输入：学校配置对象
- 输出：`Future<void>`
- 逻辑：
- 通过 `AlertDialog` 展示 id、initialUrl、targetUrl、extractScriptUrl。

### 4.3 课程页 FAB `onPressed`
- 输入：点击事件
- 输出：导航行为
- 逻辑：
- `Navigator.push` 到 `ImportSchoolListPage`。

## 5. 页面结构说明
### 5.1 导入学校列表页
- 顶栏：`Import Schedule`
- 内容区：
1. 说明卡片（来源说明）
2. 状态区（加载中 / 失败 / 空列表 / 正常列表）
3. 学校条目卡片列表

### 5.2 交互状态
1. 首次加载：`initState -> _load()`
2. 下拉刷新：`RefreshIndicator`
3. 失败重试：`Retry` 按钮
4. 条目点击：配置预览弹窗

## 6. 架构设计说明
1. 页面仅负责状态与展示，不直接写 HTTP。
2. 数据读取通过 `SchoolConfigRepository`，保持分层边界。
3. 课程页仅负责导航，不承载导入列表逻辑。

## 7. 执行方案与关键决策
1. 决策：先做学校列表，不直接实现 WebView 导入。
- 原因：先打通“入口 + 配置可见性”，降低复杂度。

2. 决策：增加配置预览弹窗。
- 原因：方便开发阶段核对配置下发内容。

3. 决策：保留错误可见与重试。
- 原因：后端不可用时也能定位问题，不做“静默失败”。

## 8. 验收标准
1. 课程页点击右下加号可进入导入学校列表页。
2. 学校列表页可以触发配置加载。
3. 网络失败时可显示错误并支持重试。
4. 条目点击可查看配置关键字段。
5. `flutter analyze` 通过，无静态错误。

## 9. 验收结果（本次）
1. 导入学校列表页已创建并接入入口。
2. 课程页 FAB 已接入导航。
3. 静态检查通过：`No issues found`。
