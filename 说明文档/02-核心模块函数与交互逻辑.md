# 核心模块函数与交互逻辑

## 1. 课程表模块（Schedule）
### 1.1 `_openAddMenu()`
- 文件：`classduck_app/lib/features/schedule/ui/schedule_page.dart`
- 作用：打开右下角列表浮层菜单（手动添加/教务添加/拍照添加）。
- 关键逻辑：
1. 使用 `showGeneralDialog` 创建右下角浮层。
2. 菜单项点击返回 action。
3. 按 action 分发到对应二级流程。

### 1.2 `_openScheduleSidebar()`
- 文件：`classduck_app/lib/features/schedule/ui/schedule_page.dart`
- 作用：打开课表左侧 7 分组抽屉面板（课表设置/时间设置/上午课程/下午课程/晚上课程/周数设置/开学日期）。
- 关键逻辑：
1. 使用 `showGeneralDialog` + 左滑入动效，背景含灰色遮罩和模糊。
2. 按 `_expandedSidebarGroup` 控制单组展开/折叠。
3. 课表设置分组支持新建课表和多课表切换。
4. 上课时间/分段/周数/开学日期修改后，实时更新主页面时间轴与周视图。

### 1.3 `_switchCourseTable(tableId, tableName)`
- 作用：切换当前激活课表并刷新课程数据。

### 1.4 `_ScheduleConfig`
- 文件：`classduck_app/lib/features/schedule/ui/schedule_page.dart`
- 作用：承载侧边栏配置（课时模板、分段节次、周数、起始日、开学日期）。
- 联动：配置变化后同步到 `_periodTimes`，并写回 `course_table.class_time_list_json`。
- 语义：上午/下午/晚上已改为“课程节数”语义，使用 `morningCount/afternoonCount/eveningCount` 推导节次区间。

### 1.5 `pickTime(index, start)`
- 作用：修改单节次开始或结束时间。
- 联动规则：
1. 调整开始时间时，该节结束时间按课时时长自动重算。
2. 调整当前节次后，后续节次按“上一节结束 + 课间 + 课时”顺延。
3. 当开始时间不早于结束时间时，阻止保存并提示。

### 1.6 `_ScheduleConfig.alignByTemplate()`
- 作用：执行“一键批量对齐”。
- 当前规则：按上午/下午/晚上三个时段“分别对齐”，只在各时段内部连续；时段之间保留午休/晚休断层。

### 1.7 `_openTimeEditDialog(context, initialValue)`
- 作用：课表节次时间编辑弹窗（中文）。
- 交互规则：
1. 默认进入“手动输入模式”（小时/分钟）。
2. 右下角按钮可切换到“滚轮模式”。
3. 再次点击可切回手动输入。
4. 输入值做范围校验（小时 0-23，分钟 0-59）。

### 1.4 `_handleCameraAdd()`
- 作用：调用系统相机并处理取消/失败提示。
- 异常处理：权限拒绝、用户取消、设备调用失败。

## 2. 选择学校模块（Import）
### 2.1 分栏切换（专科/本科/硕士/通用）
- 文件：`classduck_app/lib/features/import/ui/import_school_list_page.dart`
- 交互：点击切换 + 左右滑动切换。
- 视觉：宽胶囊 80 宽，激活态黄色背景。
- 数据：已改为真实层级过滤（按 `SchoolConfig.level` 过滤当前分栏）。

### 2.1.1 分栏口径更新（2026-03-15）
- 分栏由 4 个调整为 3 个：`本/专科`、`硕士`、`通用`。
- `junior`（专科）在模型层统一归并到 `undergraduate`，前端不再单独展示“专科”分栏。
- `通用`分栏仅展示“教务系统入口”（例如 URP/树维/青果），不直接平铺学校名。

### 2.1.2 学校配置离线兜底
- 文件：`classduck_app/lib/features/import/data/school_config_repository.dart`
- 行为：优先请求 `/v1/config/schools`；接口不可用时自动读取本地内置 `assets/config/schools.builtin.json`。
- 目的：未启动后端时也能在本地完成导入列表联调。

### 2.1.3 字母导航条手势
- 文件：`classduck_app/lib/features/import/ui/import_school_list_page.dart`
- 行为：支持点击字母与纵向滑动选字母，当前激活字母用黄色容器高亮。
- 目的：快速跳转到目标首字母分组。

### 2.2 `_jumpToLetter()`
- 作用：点击右侧字母导航后滚动到对应分组锚点。

### 2.3 `_filteredConfigs()`
- 作用：执行“层级筛选 + 关键词筛选”的组合过滤。
- 说明：先按专科/本科/硕士/通用筛一遍，再叠加搜索关键字。

### 2.4 `AdapterRuleRepository.fetchRules()`
- 文件：`classduck_app/lib/features/import/data/adapter_rule_repository.dart`
- 作用：读取后端 `/v1/config/adapters` 规则列表，为后续解析器匹配做准备。

### 2.5 `ImportExecutionPage`
- 文件：`classduck_app/lib/features/import/ui/import_execution_page.dart`
- 行为：
1. 支持编辑“登录页URL”。
2. 提供“打开登录页”按钮，先完成账号登录。
3. 再点击“下载并导入课表”执行导入流程。

## 3. 待办模块（Todo）
### 3.1 `_openSidebarPlaceholder()`（已升级为任务类型管理）
- 文件：`classduck_app/lib/features/todo/ui/todo_page.dart`
- 作用：管理任务类型，支持新增自定义类型。
- 联动：新增类型后同步影响分组展示逻辑。

### 3.2 `_openSidebarPlaceholder()`（当前为完整侧边栏）
- 文件：`classduck_app/lib/features/todo/ui/todo_page.dart`
- 作用：打开待办左侧抽屉，展示类型卡片与 `+ 新建类型` 入口。
- 关键逻辑：
1. 抽屉内通过卡片入口触发 `_createNewType()`。
2. 新增后同步写入 `_typeOrder/_typeLabels`。
3. 通过遮罩点击可关闭抽屉。

### 3.2.1 侧栏状态机（按 PRD）
- `sidebar_open_normal`：仅展示分组与 `+ 新建待办`。
- `sidebar_open_editing`：插入“自定义名称 + 勾叉”输入行。
- 关闭侧栏：若仍在编辑态，按取消处理（不写入）。
- 长按自定义类型：进入删除待确认态，可移除自定义类型。

### 3.3 `_createNewType()`
- 作用：弹窗输入任务类型名称，完成唯一性检查后创建自定义类型。

### 3.4 `TodoRepository.deleteTodosByTaskType()`
- 文件：`classduck_app/lib/features/todo/data/todo_repository.dart`
- 作用：按类型批量删除待办，供“删除待办类型”联动清理使用。

### 3.5 `TodoPage._loadCourseOptions()`
- 文件：`classduck_app/lib/features/todo/ui/todo_page.dart`
- 作用：从课表真实课程动态聚合课程名称，供新建待办“关联课程”模块使用。

### 3.6 `TodoPage` 折叠策略
- 作用：待办分组默认折叠，未展开时不显示任务项。

### 3.7 `TodoPage._openNewTodo()`
- 作用：点击新建待办前，先实时重新读取课表课程列表，再传入新建页。
- 结果：降低“刚新增课程但新建待办里暂时看不到”的延迟感。

### 3.3 `_toggleCompleted()`
- 作用：切换任务完成状态并刷新分栏。
- 结果：未完成与已完成之间迁移。

## 4. 新建待办模块（NewTodo）
### 4.1 `NewTodoPage(taskTypeLabels, courseOptions)`
- 文件：`classduck_app/lib/features/todo/ui/new_todo_page.dart`
- 作用：接收待办主页传入的动态类型和课程列表，按当前数据状态渲染。

### 4.2 `_buildTaskTypePills()` / `_openTaskTypeSelector()`
- 作用：任务类型默认最多展示 3 个，超出后用 `...` 入口弹出完整类型列表。

### 4.3 `NewTodoPage._submit()`
- 作用：关联课程改为可选项，保留任务内容必填。

### 4.2 `_submit()`
- 文件：`classduck_app/lib/features/todo/ui/new_todo_page.dart`
- 必填校验：计划内容；关联课程仅在存在课程选项时必填。
- 成功后：写入待办数据并返回上页触发刷新。

### 4.3 `_pickDate()` / `_pickTime()`
- 作用：分别控制截止日期与截止时间。
- 交互：已改为底部弹出旋轴（`CupertinoDatePicker`）滚轮选择，含取消/确定动作。

### 4.4 `_showWheelPicker()`
- 作用：统一封装日期/时间滚轮弹窗，减少重复 UI 代码。

### 4.5 `_inputCustomCourse()`
- 作用：通过弹窗输入“更多”课程选项。

### 5.1 `_openDoneCourseModal()`
- 文件：`classduck_app/lib/features/profile/ui/profile_page.dart`
- 作用：打开“已上课程”只读弹窗。

### 5.4 `_showReadonlyGroupedModal()`
- 作用：统一封装“可搜索 + 分组抽屉 + 只读列表”的统计弹窗骨架。

## 6. 设置模块（Settings）
### 6.1 `ReleaseRepository.checkRelease()`
- 文件：`classduck_app/lib/features/settings/data/release_repository.dart`
- 作用：调用后端 `/v1/release/check` 获取版本更新信息。

### 6.1.1 强制更新接口预留
- 后端入口：`/v1/release/check`
- 预留策略：由后端配置决定是否“强制更新”，客户端按返回字段执行弹窗与拦截（本轮先保留接口与文档，不启用自动拦截逻辑）。

### 6.2 `AboutPage._loadVersionCheckResult()`
- 文件：`classduck_app/lib/features/settings/ui/about_page.dart`
- 作用：优先读取后端版本检查结果，失败时走本地兜底数据。

### 6.3 `AppearancePage._selectMode()`
- 文件：`classduck_app/lib/features/settings/ui/appearance_page.dart`
- 作用：切换浅色/深色模式选中态并即时反馈。

### 6.4 `AppearancePage._uploadBackgroundImage()` / `_clearBackgroundImage()`
- 作用：上传或清除背景图片配置入口，维护背景图状态。

### 6.5 `AppearanceStore`
- 文件：`classduck_app/lib/features/settings/data/appearance_state.dart`
- 作用：提供全局外观状态（主题模式、背景图字节），供设置页写入、课表页监听。
