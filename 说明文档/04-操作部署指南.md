## 2. Python 导入后端

### 2.1 本地开发

```bash
cd backend/python_import_service

# 创建虚拟环境（首次）
python -m venv .venv

# 激活环境
# Windows:
.venv\Scripts\activate
# macOS/Linux:
# source .venv/bin/activate

# 安装依赖
pip install -r requirements.txt

# 启动开发服务器（带热重载）
uvicorn main:app --reload --host 0.0.0.0 --port 8000
```

启动后访问：
- 健康检查：`http://localhost:8000/health`
- API 文档：`http://localhost:8000/docs`（FastAPI 自动生成 Swagger UI）
- 学校列表：`http://localhost:8000/v1/config/schools`

### 2.2 运行测试

```bash
cd backend/python_import_service
pytest tests/ -v
```

当前覆盖 6 个测试用例：
1. `test_list_schools_v1` — 学校列表 v1 兼容格式
2. `test_get_school_config` — 学校配置详情
3. `test_get_provider_script` — JS 脚本下载
4. `test_validate_import_success` — 导入校验成功（2 门课，含范围周 + 双周）
5. `test_validate_import_invalid_json` — 导入校验失败（无效 JSON）
6. `test_report_log_v1` — v1 格式日志上报

### 2.3 Docker 部署

```bash
cd backend/python_import_service

# 构建镜像
docker build -t classduck-import-service .

# 运行容器
docker run -d -p 8000:8000 --name classduck-import classduck-import-service
```

> 注意：Dockerfile 当前基于 `python:3.11-slim`。若安全扫描报告基础镜像漏洞，
> 可切换为 `python:3.12-slim` 或 Alpine 变体。

### 2.4 新增学校

1. 在 `data/school_configs/` 下新建 `{school_id}.json`：
   - 参照 `xjtu.json` 格式填写 `field_mapping`、`fetch_config`、`timer_config` 等
   - `id` 字段必须与文件名（不含扩展名）一致
2. 在 `data/scripts/` 下新建 `{school_id}_provider.js`：
   - 脚本在 WebView 内执行，通过 `window.flutter_inappwebview.callHandler` 回传数据
   - 支持 `{{YEAR}}` / `{{TERM}}` 占位符
3. 重启服务：新增的配置和脚本会在启动时自动加载

### 3.3 导入链路双通路说明

Flutter 导入执行页 (`import_execution_page.dart`) 支持两条导入通路：

**新通路（推荐）—— 后端校验**
1. WebView 打开教务登录页
2. 用户登录后，注入 JS 脚本获取原始课表 JSON
3. `_rawJsonFromJs` 存入原始数据
4. 点击导入 → `ImportEngine.importFromRawJson()` → POST `/api/import/validate` → 存入 SQLite

**旧通路（兜底）—— 本地解析**
1. WebView 打开教务登录页
2. 用户登录后，抓取 HTML 页面源码
3. `_capturedHtml` 存入 HTML
4. 点击导入 → `ImportEngine.importFromCapturedHtml()` → Dart 本地 parser → 存入 SQLite

选择逻辑：`_rawJsonFromJs` 有值则走新通路，否则走旧通路。

### 3.4 脚本来源与 SORA 显示规则

导入页抓取脚本支持两级来源：

1. 远端脚本（优先）
- 请求：`GET /api/schools/{schoolId}/script?script_type=provider`
- 用途：使用后端动态下发的学校脚本。

2. 本地脚本（回退）
- 当远端脚本请求失败时，若学校配置中的 `extractScriptUrl` 是 `local://` 开头，则自动映射到 Flutter 资源。
- 映射规则：`local://parsers/sora_provider.js` → `assets/parsers/sora_provider.js`

SORA 适配项默认已包含：
- 后端配置：`backend/python_import_service/data/school_configs/sora.json`
- 后端脚本：`backend/python_import_service/data/scripts/sora_provider.js`
- 前端回退脚本：`classduck_app/assets/parsers/sora_provider.js`

若出现“看不到 SORA”通常排查以下两点：
- 当前学校分栏是否在“通用”分组（可切换到“通用”或直接搜索 `sora`）
- 后端是否已启动并成功加载 `sora.json`

### 3.5 大学课程导入（豆包链路）

新增了一条不依赖教务 WebView 的截图导入链路，适用于“已经拿到课表截图，只想快速导入”的场景。

入口位置：

1. 课程表页
2. 点击右下角 `+`
3. 选择 `大学课程导入`

页面内操作：

1. 点击“一键复制提示词并跳转豆包”
2. 在豆包中选择专家模型
3. 把课表截图和提示词一起发送
4. 复制豆包返回结果
5. 回到 App 粘贴结果并导入

导入约束：

1. 豆包返回内容必须是 JSON；当前前端支持纯 JSON、带 ```json 代码块的 JSON，以及顶层 JSON 数组。
2. 当前优先兼容的字段契约为：`n`、`d`、`s`、`e`、`w`、`l`、`t`。
3. 同时兼容旧长字段格式，例如：`name`、`day`、`start_section`、`duration`、`weeks`。
4. 课程数组字段至少要能解析出：课程名、周次、星期、开始节次、结束节次或连续节数。
5. 周次支持数组和字符串两种格式，例如：`[1,2,3]`、`1-16周`、`1-16周(单)`、`1-8,10,12`。
6. 导入后若已有课表，仍然会弹出“新建课表 / 覆盖当前课表”的冲突决策。

导入后仍需用户自行确认：

1. 总节数，例如 14 节
2. 学期周数，例如 25 周
3. 每节课时间；若各节时长不一致，请关闭统一时长后逐节调整

## 4. 端到端联调步骤

1. **启动 Python 后端**
   ```bash
   cd backend/python_import_service
   .venv\Scripts\activate
   uvicorn main:app --reload --host 0.0.0.0 --port 8000
   ```

2. **验证后端**
   ```bash
   # 健康检查
   curl http://localhost:8000/health
   # 学校列表
   curl http://localhost:8000/v1/config/schools
   ```

3. **启动 Flutter**
   ```bash
   cd classduck_app
   flutter run -d chrome    # 或其他目标
   ```

4. **执行导入**
   - 课程表页 → 点击加号 → 教务添加
   - 选择学校（如 XJTU）
   - 进入导入执行页 → 打开登录页 → 登录
   - 点击下载按钮执行导入

5. **执行大学课程导入（豆包）**
   - 课程表页 → 点击加号 → 大学课程导入
   - 复制提示词
   - 打开豆包并选择专家模型
   - 发送课表截图 + 提示词
   - 复制结果 → 回到 App 粘贴 → 导入

6. **检查日志**
   ```bash
   curl http://localhost:8000/api/import/log   # 查看是否有日志写入
   ```

## 5. 注意事项

1. **后端先于前端启动**：Flutter 启动时不会主动连后端，但导入、学校列表等功能依赖后端在线。学校列表有本地 builtin 兜底。
2. **WebView 仅原生端可用**：Web 端无法使用内嵌 WebView，会提示不可用。
3. **数据库文件位置**：
   - Python 日志 DB：`backend/python_import_service/data/db/logs.db`（自动创建）
   - Flutter 课表 DB：由 sqflite 管理，在设备应用沙箱内
4. **端口冲突**：若 8000 被占用，改用其他端口并同步修改 `AppEnv.apiBaseUrl`。
5. **CORS**：开发阶段允许全部来源；生产部署必须收紧 `allow_origins`。
