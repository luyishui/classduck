## 5. 教务导入重建方案（核心）
### 5.1 流程（双通路）

**新通路（Python 后端校验 —— 推荐）**
1. 拉取学校配置 `GET /api/schools/{id}/config`
2. 下载 JS 注入脚本 `GET /api/schools/{id}/script`
3. WebView 打开登录页，监听导航
4. 到达目标页后注入 JS 脚本，脚本回调返回原始 JSON
5. 将 rawJson 发到后端 `POST /api/import/validate` 做字段映射、周次/节次解析与标准化
6. 后端返回标准化 `courses[]`，前端写入本地 SQLite
7. 上报导入日志 `POST /api/import/log`
8. 刷新 UI 与统计

**旧通路（Dart 本地 HTML 解析 —— 兜底）**
1. 拉取学校配置（含 initialUrl、targetUrl、delay）
2. WebView 打开登录页，监听导航
3. 到达目标页后抓取 HTML
4. Dart 侧 `XjtuScheduleHtmlParser` 本地解析
5. 解析结果标准化、落库
6. 刷新 UI 与统计

`ImportExecutionPage` 根据 `_rawJsonFromJs` 是否有值自动选择通路。

### 5.2 数据契约（统一）
导入脚本返回：
- `name`: 学期名
- `courses[]`: 课程数组（课程名、地点、教师、周次、星期、开始节、持续节数等）

应用内统一映射为：
- `ImportedTable`
- `ImportedCourse`

### 5.3 可扩展设计
- 适配器接口：`ScheduleProviderAdapter`
- 解析器注册中心：`ParserRegistry`
- 学校配置版本化与缓存：支持在线失败回退本地缓存
